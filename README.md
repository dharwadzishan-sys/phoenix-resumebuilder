# 🚀 Phoenix AI Interview Suite

Phoenix is a cutting-edge, AI-powered interview preparation platform designed to help candidates master their technical and behavioral interviews. Using advanced LLMs (Gemini & Groq), it provides real-time feedback, ATS-optimized resume building, and personalized study roadmaps.

---

## ✨ Key Features

- **🎯 AI Interview Engine**: Interactive voice and text-based interview simulations with real-time AI evaluation and scoring.
- **📄 Pro Resume Builder**: Create ATS-optimized resumes with AI-generated professional summaries and skill highlights.
- **🗺️ Smart Study Roadmap**: Personalized learning paths generated by AI based on your target role and experience level.
- **💬 Real-time Q&A**: Intelligent question-and-answer module to clarify complex technical concepts instantly.
- **📹 Integrated Media**: Support for camera and microphone interaction to simulate real-world interview conditions.

---

## 🛠️ Tech Stack

- **Frontend**: [Next.js 15+](https://nextjs.org/), [React 19](https://react.dev/), [Tailwind CSS](https://tailwindcss.com/), [Framer Motion](https://www.framer.com/motion/)
- **Backend**: [Node.js](https://nodejs.org/), [Express.js](https://expressjs.com/)
- **Database & Auth**: [Firebase](https://firebase.google.com/)
- **AI Integration**: [Groq SDK](https://groq.com/), [Google Gemini API](https://ai.google.dev/)
- **State Management**: React Context API

## 🚀 Getting Started

### 1. Prerequisites
This project requires **Node.js v20.9.0+**. 
*(Note: A local Node v20 binary is included in the `.nodelocal` folder for Windows users to ensure compatibility.)*

### 2. Installation
Clone the repository and install dependencies in the root folder:
```bash
git clone https://github.com/dharwadzishan-sys/phoenix-resumebuilder.git
cd phoenix-resumebuilder
npm install
```

### 3. Environment Setup
Create a `.env.local` in the root and a `.env` in the `/backend` folder using the provided `.env.example` files as templates.

### 4. Running the Project
The project uses a unified command to start both the frontend and backend simultaneously:
```bash
npm run dev
```
- **Frontend**: http://localhost:3000
- **Backend**: http://localhost:5000

---

## 📁 Project Structure

- `/src`: Next.js frontend application (App Router).
- `/backend`: Node.js Express server handling AI logic and Firebase integration.
- `/public`: Static assets and images.

---

## 🔒 Security
- Environment variables are strictly ignored via `.gitignore`.
- Firebase service account keys are never committed to the repository.
- Ensure you use your own API keys for Groq and Firebase in production.
